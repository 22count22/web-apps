The What and the Why

NodeJS is Javascript on the server (also known as the "backend"). Many clients, such as your mobile phones, talk to a central server, such as Google or Facebook.

When you upload pictures to the cloud, servers are what processes and stores them.

If front-end applications (like the ones you build in your browser) are the restaurant floor, servers are the back kitchen.

Why Node in the Server?

We already know Javascript!

Node is fast, and built for the web.

Has some of the best support for libraries.

You want to resize and image? There is a library.

You want to encode video? There is a library.

You want to make a cow say ridiculous things? There. Is. A. Library.

fs - working with the (f)ile (s)ystem


(pair programming) 
            
// colored code snippets that don't run
const fs = require('fs')

fs.readFile('./my_special_text.txt', (err, data) => {
  if (err) {
    console.log(err)
    return
  }

  console.log(data)
})

fs.writeFile('./my_special_message.txt', 'Hello Node.js', (err) => {
  if (err) {
    console.log(err)
    return
  }
  console.log('The file has been saved!');
});

Motivation for fs

Files let you store data long-term. We know this. But learning to work with files and manipulate file contents in code gives you the ability to write real life applications.

  • Generate reports.
  • Process a document to remove bad words.
  • So much more!
